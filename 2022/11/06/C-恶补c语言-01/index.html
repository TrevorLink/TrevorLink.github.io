变量定义

有两种风格的定义，主要在于变量定义的位置不同。

  • C99 支持在程序任何位置定义变量，只要你在使用变量之前定义就可。
  • ANSI C 仅支持在程序开始时定义变量。

关于scanf和printf的误区

这两个的f其实都是format的意思，分别都是格式化的输入和输出。格式化输出好理解，但是格式化输入其实是严格要求输入的格式的，因此最好别在scanf中出现空格和\n这种。

常量相关

直接写在代码中的魔法值(Magic Number)叫做直接量(literal)，一般来说我们更喜欢通过一个const修饰符来进行常量的声明。

有何不同

  • 可以表示的数据范围不同，对于整数型，char < short < int < long < long long
  • 输出时候的格式化要求不同，比如%d,%f等。
  • 内存中所占据的大小不同，4字节，8字节等。
  • 内存中的表达形式不同，对于基础整型数据内存的具体表示为二进制，但是对于浮点数数据比如 double 这种实际上是通过编码的形式来表达的（这一点在CSAPP中提到的浮点数IEEE标准中也是对应的）。

函数

函数原型

我们之前一般的习惯是说对于函数原先定义的顺序是 从上到下 进行定义的，也就是先声明函数（签名）。

这主要是出于为编译器的考量，因为程序是自上而下进行执行的，在编译阶段也是如此，但是这样的可读性往往很差。

我们当然也可以把 main 函数定义在最开始，读者只关心主要逻辑，了解了主要逻辑之后再深入了解其余的函数逻辑，但是这在某些编译器上是可以，某些编译器上是不可以通过的，这是因为 compiler 不同，针对 C99 的执行力度也就不同。

这个时候往就需要我们使用函数原型来解决这个问题。

函数原型的本质思想其实就是在程序的一开始就把函数的签名给定义出来，这个时候这个签名就叫做函数的原型。

值得注意的是函数的原型声明其实从理论上来说我们是不用关心参数的名称的，换句话说可以省略。但是出于于可读性，为了满足见名知意，所以我们一般在程序中定义函数的原型的时候还是会把参数保留下来。
