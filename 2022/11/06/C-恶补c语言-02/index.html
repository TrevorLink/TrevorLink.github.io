不同机器上的占用

注意 c 的指针在不同字长的机器上所占的字节数是不一样的，究竟占用多少字节取决于你的机器的架构，这个其实在我们之前和 CSAPP 的时候也都有提到过。

在 32 位机器上一个指针所占用的字节数是 4byte，而在 64 位机器上一个指针所占用的字节数是 8byte。（而int无论在64还是32位架构的机器上都是 4byte 的大小，因此我们不能简单地通过 %x的方式来进行打印，指针有专门的占位符打印方式:%p

为什么scanf直接传数字不会报错

综合上述的所有我们已经可以回答了，可能你是 32 位系统的机器，因此 int 和 pointer 变量都占用了4byte 的数据，所以在编译阶段是没有问题的，在执行 scanf 这个函数进去的时候也是没有问题， scanf 这个函数把传入的普通数字也直接当成了地址值，但是如果我们传入的是内存低地址区域的数字，那部分是不让访问的，因此会直接报错出现 segementation fault 这个错误，当然也可能不出现错误。

为什么需要指针

主要是有如下的两个场景：

  • 函数的返回值默认是只能返回一个，但是当一个返回不够，就得通过指针传递的方式通过引用间接返回多个结果数据。
  • 异常标识返回，需要返回是否正确，同时也要返回结果（在底层操作文件的库中很常见），这个在现在的 cpp/java 中不常见，取而代之的是异常 Exception 的处理

空指针

指针变量在没有初始化的时候不能使用，因为没有初始化的时候，默认值可能被分配到了一个奇怪的值，这个奇怪的值是地址值，然后你试图通过 *p 向这个地址写入数据，很有可能出现 segement fault 这样的问题。
