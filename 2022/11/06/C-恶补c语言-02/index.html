数组

定义

还是标准不一样，C99之前，数组大小只能是 const 定义，但是 C99 之后，数组的大小可以通过变量来代替。

在 32 位机器上一个指针所占用的字节数是 4byte，而在 64 位机器上一个指针所占用的字节数是 8byte。（而int无论在64还是32位架构的机器上都是 4byte 的大小，因此我们不能简单地通过 %x的方式来进行打印，指针有专门的占位符打印方式:%p

为什么scanf直接传数字不会报错

综合上述的所有我们已经可以回答了，可能你是 32 位系统的机器，因此 int 和 pointer 变量都占用了4byte 的数据，所以在编译阶段是没有问题的，在执行 scanf 这个函数进去的时候也是没有问题， scanf 这个函数把传入的普通数字也直接当成了地址值，但是如果我们传入的是内存低地址区域的数字，那部分是不让访问的，因此会直接报错出现 segementation fault 这个错误，当然也可能不出现错误。

为什么需要指针

主要是有如下的两个场景：

  • 函数的返回值默认是只能返回一个，但是当一个返回不够，就得通过指针传递的方式通过引用间接返回多个结果数据。
  • 异常标识返回，需要返回是否正确，同时也要返回结果（在底层操作文件的库中很常见），这个在现在的 cpp/java 中不常见，取而代之的是异常 Exception 的处理

空指针

指针变量在没有初始化的时候不能使用，因为没有初始化的时候，默认值可能被分配到了一个奇怪的值，这个奇怪的值是地址值，然后你试图通过 *p 向这个地址写入数据，很有可能出现 segement fault 这样的问题。

数组与指针

数组作为其他函数的参数的时候本质上传递的是一个指针。

这也就是为什么我们说传参数的时候再用 size_of 其实是求不出来的， 因为即使你用 size_of 去获取大小，获取到的本质上还是 pointer 的类型大小所占字节。（也就是说在 32 位机器上是 4 字节，在 64 位机器上是 8 字节。）

数组变量是一个特殊的指针。

数组变量本身就表达了地址，这是因为我们之前说的，数组变量被数组的第一个元素所代表，相当于起了一个别名(alias)。

  • 因此 [] 运算符也可以对指针做。
  • 因此 * 运算符也可以对指针做，表示的是获取数组指针的所指向的数组的第一个元素。

但是数组的单元他是变量，是需要用 & 来获取的。

数组变量本质是 int* const 的，因此数组本身不可变。

所以是不可以出现类似 int [] a = b 这样的代码的。

指针与const

如果考虑指针的同时我们也考虑到 const 常量这一个因素的话，我们其实不难得出，指针本身可以是一个常量，而指针所指向的那个元素它也可以是一个常量，并且这两者是有本质上的区别的。

指针是const

指针不可修改。

意思就是说地址的值不可以被改变了，换句话说，q指向i这个事实不能再改变了，q记录的永远就是i的地址值，还是直接看代码。

到底怎么看

直接看 * 到底在 const 的前面还是后面。

数组与const

我们已经知道数组作为函数的参数进行传递的时候本质上是一个 const 类型的指针变量，但是如果我们在定义的时候再加上 const 进行声明，则表示说，这个数组的每一个元素都是不可变的。但是与此同时我们也只能通过显式声明的方式来进行元素的初始化。

通过这种方式可以保护数组即使是传递参数的时候，其内部元素的 value 也不会被破坏。

同理指针-1这个运算也是让指针移动到下一个数据单元的地址。

0地址

在多进程的操作系统架构下，所有的进程都以为他们自己有从0开始一片连续内存空间（操作系统的虚拟内存地址）。

0地址一般不让碰，因此我们程序中的指针不应该具有0值，我们可以在如下的两个基本场景中使用0地址的指针：

  • 表示你返回的指针是无效的。
  • 初始化指针的时候可以先是0，之后如果指针没有再被初始化，程序就一定会报错，我们一般用 NULL 来表示0地址，因为有的 compiler 它对于数字0作为地址 和 NULL 作为地址好像还是不一样的。

问题

常见的关于 malloc 这个函数可能出现的问题如下：

  • 与 malloc 函数对应的 free 函数在释放的时候传递的指针注意不能进行数值运算，也就是说需要还一模一样的内存地址区域（即 malloc 返回的指针不能改变）
  • 当 malloc 已经没有内存可以分配的时候，此时会返回 0 或者是 NULL（一个意思），可能会导致 0 地址的问题。但是我们之前也提到过指针一般初始化的时候都习惯初始为 0 这个值，就是为了让你没有操作的时候触发 0 地址。
